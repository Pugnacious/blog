Accessing Parameters:

This guide contains eight example flames, including the exact Grand Julians used in Julia Uncovered v.2.2. All of the examples appear in the guide.

In the main Apophysis window, go to File -> Open and search for the .flame parameters file within the cj_julia_uncovered_2_2 ZIP folder. Click on the CJ_Julia_Uncovered_2_2.flame file and press enter. The flames should load into the main Apophysis window. That's it.

Please note that the .FLAME file will most likely only work with Apophysis version 2.05b2 and above as that was the application which created the Julians. If that doesn't do the trick and the flames remain invisible, it might be best to re-download Apophysis and/or talk to someone ASAP at FracFan Forums.

FAQ:

Not that they actually are, but...well...it might answer some questions you have but didn't want to ask out of fear of being rude. Or something.

So, what's different in this version of JU v.2.2?
Oh, not much. Just nearly twice the amount of information in the first official version, three total Julian techniques discussed and tons more graphics. And, it now has a version number after the title.

If you want more info on the changes, look in the Updates section above.

Grand Julians seem pretty easy to make. Why is this labeled as an intermediate guide?
Julians are very easy to create. The problem arises from the fact that Apophysis can be difficult to manipulate. For s brief moment, I considered adding a section about the Transform Editor, Options window, etc. to JU v.2.2, but decided against it since I already had created the beginner's tutorial entitled The Apophysis Guide v.2.1: A Guide to Basic Flame Creation. I didn't see the point of further enlarging JU v.2.2 when I already had something else out there.

What on earth possessed you to write a 55 page guide just for Grand Julians?
Well, technically it's not 55 pages of actual text, but I get your point.

I'm not really sure. Sometimes, I get these ideas to write something and it builds up in my brain until I have to do it. It starts out as "hmm, Julians from Apophysis 2.04b aren't importing correctly into any other version". A little later it's "boy, that layout is really starting to bug me." I guess it's also, "whoa, when I wrote Julia Uncovered I thought I knew so much when in reality I knew crap about Julians!" From there, I kept thinking of more useful information to include that would help Apophysis users. I added the mini-glossary to help less knowledgable users with some of the basic terms, though looking back at the entries I wonder if anyone is going to understand a thing I said.

I guess the real answer is that I like technical writing and I like Apophysis. I also like helping people and don't want others to be turned off by the more difficult aspects of the application. When I first came onto the glorious i-net, I used to wonder what kind of person would take the time to write tutorials, FAQs and walkthroughs. I guess that kind of person is me.

What on earth possessed you to write and code such a large HTML readme?
No comment.

What's with the smart ass readme?
Smart ass readmes always seemed like tons of fun to write.

Why didn't you use a CSS page for the readme?
My goal was to make the simplest readme as possible. In my experience, readers are often confused by HTML files and folders, which is why I use PDF files only for tutorials. There were some little details I had wanted to include, such as adding a little CJ icon to the page title, but that would have required extra graphics and possibly additional folders (and only coders like me would have noticed).

Typically I relied on TXT files in the past, but decided to use a more platform friendly format this time around (though most people who use Apophysis use Windows, so it's all kind of moot). Anyway, I got to work on coding and create a much prettier readme while only relying on one type of document.

What's up with the horizontal rules in the readme? They're like 25% of the main table.
Technically they are a third of the table's width.

I had intended to use full length rules or at least different colored cells for the section titles, but I didn't like the results. Instead I returned to the partial horizontal rule width. Trust me. They looked far less blocky than lots of rectangular boxes.

Why didn't you use the <UL> and <LI> tags for the bulleted lists?
I'm glad you asked. I thought I was the only one who would have noticed.

You see, while I designed the readme for Firefox, I realized that users would continue to use other browsers like Explorer. HTML and CSS render differently in different browsers. Some elements look better in one browser over another. The list laid out beautifully in Explorer (for some reason), but was completely wrong in Firefox. I had the same problem when developing my Apophysis website, so I took that code and applied it here. Since I do not really need a list function, I felt it was fine to fudge the coding to make it appear as though I had used the <UL> tag.

In the Transform Editor section of the guide you stated that you would refer to triangles as "transforms" yet your new screenshots continue to refer to them as "triangles". What's up with that?
Yes. I only noticed that error when I converted the files to HTML for the online version. Frankly, I'm sick of the whole thing at this point (there's a heck of a lot of data to keep tabs on) and I don't think anyone will care. In fact, the hover text in the Transform Editor refers to transforms as "triangles". Just for the record, the screenshots for Julian Variation 3 use the term "transforms".

May I upload/offer commercially/place on a website/show my girlfriend an image I created from your tutorial?
Read the opening of Julia Uncovered v.2.2 and/or Terms of Use section below. But basically, YES. Now stop asking me.

May I upload/offer commercially/place on a website/show my girlfriend an image I created from your flame pack?
Probably. As long as the alteration to the flame is more than a simple gradient change, it should be fine. Again, read the opening of Julia Uncovered v.2.2 and/or Terms of Use section below.

Is it okay if I print out Julia Uncovered?
Of course. That's why I created the tutorial as a handy-dandy PDF. But beware. Fifty-five pages of graphics will do a number on your inkjet priter. I recommend using a laserjet, if possible.

I have an image that looks kind of like yours, but I swear! I came up with it completely on my own...do I still need to credit you?
NO. If you say that the flame is entirely your own, I believe you. And anyway, most Apophysis flames look similar to someone else's work. It's just a hazard of fractals and how they are created.

Can I take JU v.2.2, decompile the contents, upload it to the Internet and claim it as my own?
Stop being smart. You already know the answer.

Would it be okay if I host JU v.2.2 on my personal website?
Absolutely. Just do not alter the ZIP contents or charge for download. Read the Terms of Use section below for more information.

Is it possible to use an original Claire Jones image for fill-in-the-blank-scenario?
Possibly, but you will need express permission first. Send an e-mail.

Um, I noticed that this version is just a smidge larger than the other one.
And your question is?

I sorta kinda have dial-up. Do you hate me?
No. I hate your crappy dial-up ISP. My recommendation is to drop it and switch to cable or DSL. Or download the file at work/school.

On a serious note, I didn't want to have so many images because I knew the file size would increase dramatically and having been a dial-up user I didn't want to inconvenience anyone. However, I strongly felt that the images were necessary for JU v.2.2 and even kicked out a lot of graphics. I tried my best to shrink the remaining images as far as possible with minimal visual loss. I'm amazed that I was able to cram nearly 90 images into 5 MB, since I would have guessed the end result to be much larger. Hopefully, I can get my Apophysis website up, which does have HTML versions of my tutorials available. It was some kind of horrible nightmare converting the text for JU v.2.2 into an online format.

What's with the deviantART color scheme for the readme?
Well, I hate picking out color palettes and this was a ton easier. You should have seen how much effort went into picking the blue link color. I'm also becoming bored with the black background typical of my HTML coding.

I did like the color schema and most people who will download JU v.2.2 are already dA users accustomed to this look. It's a win-win situation all the way around. Unless green makes you vomit, then I suppose it's a lose-lose situation, though maybe it's still a win-lose situation since I like the palette and could care less if you hate it.

Did you really figure this Apophysis/Julian stuff out all on your own?
Pretty much, though that doesn't mean that I haven't had a lot of pointers and helpful criticism given to me since I began working with Apophysis in mid-2005. Plus, I've read up on many different techniques on dA, the Internet and FracFan, where I've gleaned lots of little tips here and there. I try to credit everyone that I can, as you will see in JU v.2.2. I think the majority of us are picking it up as we go along.

Users L33tM0b1l3 and Michael Faber have written to me directly to clear up some issues I had with Filter Radius/Oversample and Julians, respectively. Users 2B2H, o-LJ-o, tdierikx, utak3r, datagram, ideviant, nemopaice and Zueuk have helped indirectly with their tutorials, flamepacks and comments.

What rendering settings do you typically use?
I generally render Two Transform flames at 1280x1024 resolution at 4000 Quality and a Filter Radius of 0.6-0.8, Oversample 2. Usually, this takes me about 12-15 minutes through Apophysis on my faster system. For renders with high saturation and hotspots, I render the same settings on flam3, which takes about 50-60 minutes.

Julians take longer to render in general and require more care. If rendering a Julian with a Transform 1 containing another variation beside Blur, I set the quality at anywhere between 8000-15,000, Filter Radius 0.4-0.8 and an Oversample 2 for a 1280x1024 resolution. This generally takes about 45-60 minutes through Apophysis.

Wow, you put a lot of effort into your tutorials and I really appreciate it. Is there any other way I can support you?
Yes. No cost options include linking to my main deviantART page or tutorials directly on deviations that reference my techniques. Adding JU v.2.1 to your list of Favorites is unnecessary. Not that I don't appreciate the gesture, more that I think faving is better suited to pretty pictures.
